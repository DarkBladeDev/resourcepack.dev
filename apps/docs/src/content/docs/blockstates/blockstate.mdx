import { JsonSchema, ApiProperty } from '@resourcepack/ui';

Blockstates dictate what model to display based on the block's current in-game properties (like facing direction, lit status, or waterlogged state).

There are two primary ways to define a blockstate: **Variants** and **Multipart**.

## Variants Example

```json
{
  "variants": {
    "facing=north": { "model": "block/furnace" },
    "facing=south": { "model": "block/furnace", "y": 180 },
    "facing=west":  { "model": "block/furnace", "y": 270 },
    "facing=east":  { "model": "block/furnace", "y": 90 }
  }
}
```

<JsonSchema>
  <ApiProperty name="variants" type="object">
    Maps property string permutations to model configurations.
  </ApiProperty>
  <ApiProperty name="multipart" type="array">
    A list of conditional cases that combine multiple models simultaneously. Used heavily for blocks like fences and walls.
  </ApiProperty>
</JsonSchema>
