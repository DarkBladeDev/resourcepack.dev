import { JsonSchema, ApiProperty, Warning } from '@resourcepack/ui';

Every Resource Pack must contain a `pack.mcmeta` file in its root directory. Without it, Minecraft will not recognize the pack.

<Warning>
The `pack_format` integer changes frequently between minor Minecraft updates. Using an incorrect format will cause the game to display a warning that the pack is "incompatible", although it may still load successfully depending on the contents.
</Warning>

## Basic Structure

```json
{
  "pack": {
    "pack_format": 32,
    "description": "My Custom Resource Pack"
  }
}
```

## Properties

<JsonSchema>
  <ApiProperty name="pack" type="object" required>
    The root object for the pack metadata.
  </ApiProperty>
  <ApiProperty name="pack.pack_format" type="integer" required>
    The version format this pack was built for.
  </ApiProperty>
  <ApiProperty name="pack.description" type="string | Rawtext" required>
    The description text shown in the Resource Pack selection menu. Supports JSON rawtext components for colors.
  </ApiProperty>
  <ApiProperty name="pack.supported_formats" type="object | array | integer">
    Specifies a range of acceptable pack formats to avoid the "incompatible" warning for multi-version packs.
  </ApiProperty>
</JsonSchema>
